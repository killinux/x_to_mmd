# CLAUDE.md — x_to_mmd 强约束(HARD CONSTRAINTS)

> 本文件是开发本插件时的**强约束**,**优先级高于 [PLAN.md](PLAN.md) 里的一般做法**。
> 任何实现/重构都必须遵守;若与 PLAN/RESEARCH 冲突,以本文件为准。

## 1. 权重(WEIGHTS)

- **【强约束】尽量复用 XPS 现有权重。** 整个转换过程中,优先**保留并迁移** XPS 模型自带的顶点权重——XPS 权重通常来自游戏引擎、质量好。
- **【强约束】不到万不得已不要自行切分权重。** 不要主动把一根骨的权重按几何平面"切"给多根新骨。
  - 需要"新骨有权重"时,**首选**:把已有 XPS 骨**映射/重命名/合并**成目标 MMD 骨,让现有权重**直接跟过去**;或把现有骨的整组权重**整体迁移**到目标骨。
  - 只有当 MMD 结构**必需**某根骨、且**确实没有任何现有 XPS 权重可复用**时,才作为**最后手段**切分权重,并在代码/日志里**明确记录原因**。
  - 典型:`上半身/下半身` 的腰部分叉——优先复用 XPS 里已分别承担上身/胯部的骨,**不要**默认按水平面切骨盆顶点权重(该做法降级为 last-resort)。
- **【强约束·必须确认】所有切分权重的操作都要先找用户确认(human-in-the-loop)。** 任何会拆分/重分配顶点权重的步骤,执行前必须**暂停**并向用户说明:哪根骨、为什么必须切、打算怎么切;得到**明确同意**后才执行。未确认 → **不切**。这条对自动/一键流程同样适用(遇到要切权重就停下来问)。
- **【不变量】改名/合并时权重必须原子携带、不丢失**:网格侧顶点组先 改/合并 → 再改骨名 → 再刷 depsgraph;删骨前先把其顶点组权重 **ADD 进保留骨**,再删。
- **【不变量】控制骨**(全ての親 / センター / グルーブ / 腰 / 足ＩＫ / つま先ＩＫ / 足IK親 / 肩P 等)一律**空权重 + `use_deform=False`**。
- 导出前 `Limit Total = 4` + 重归一(XPS≤4 骨/顶点,PMX BDEF4 上限 4)。

## 2. 骨骼补全与权重检查(BONE COMPLETION)

- **【事实】XPS 骨架通常不全** —— 常缺 `全ての親 / センター / グルーブ / 上半身2 / 目 / 足ＩＫ・つま先ＩＫ / 足D・ひざD・足首D / 腕捩・手捩 / 肩P` 等;有时连独立的 `下半身` 都没有。
- **【强约束】补全骨骼极易丢权重,必须单独检查和处理。** 在任何"补全 / 插入 / 重命名 / 合并 / 重新父子化"骨骼的步骤**前后**,对每个 deform 网格做**权重完整性检查(weight-integrity check)**:
  - 快照对比:每顶点权重总和(应守恒)、每个顶点组的总权重与受影响顶点数、有无顶点组被清空 / 孤立、有无"零权重顶点"。
  - 补全后若出现:某顶点总权重下降、某组权重凭空减少、某组变空、或网格出现无权重顶点 → **报警并停下来**(不要静默继续)。
- **【强约束】新补的骨没有现成权重可继承时,不要擅自给它造权重。** XPS 缺的骨往往没有对应顶点组;遇到"这根新骨该不该有权重、权重从哪来":能用**现有骨映射/整体迁移**解决就用;若只能**切分** → 走 §1 的**用户确认**流程。
- **配套实现:** 提供可复用的 `weight_integrity_check(before, after)` 工具,贯穿 补全 / IK / 扭转 / D骨 各步骤前后调用;作为 P0"权重安全转移模块"的一部分。

## 3. 影子 / 虚拟骨(_shadow_ / _dummy_)与付与(APPEND TRANSFORM)

- **【事实】`_shadow_` / `_dummy_` 是 mmd_tools 在视口里模拟"付与(additional transform)"的内部辅助骨,不是导出数据。** PMX 导出器靠 `is_mmd_shadow_bone=True` 跳过它们;真正写进 PMX 的付与来自源骨的 `mmd_bone.additional_transform_*`。
- **【强约束】不要手动创建 `_shadow_` / `_dummy_` 骨**(CTMMD 的错:手建却不设 `is_mmd_shadow_bone` → 可能被当真骨导出、与规范生成的重复并存、且付与字段为空导致导出 PMX 没有付与)。
- **【强约束】只设付与"真字段",由 mmd_tools 生成影子骨。** 流程(放在末端,所有真实骨就位后):
  1. 对每个付与骨设 `mmd_bone.has_additional_rotation/location=True`、`additional_transform_bone=<源骨>`、`additional_transform_influence=<系数>`(腕捩/手捩 0.25/0.5/0.75、足D/ひざD/足首D=1.0、肩C=-1.0)、`transform_order=1`(源骨留 0,保证"源先变形、付与后跟随");
  2. 调 `bpy.ops.mmd_tools.apply_additional_transform()`(= `FnBone.apply_additional_transformation()`)让 mmd_tools **规范生成** `_shadow_/_dummy_`(自带 `is_mmd_shadow_bone`、正确父子/roll、`use_deform=False`);
  3. 再 `convert_to_mmd_model` → 导出。
- **【强约束】影子/虚拟骨一律 `use_deform=False`、零权重,不参与权重迁移,也绝不触发权重切分**;`weight_integrity_check` 必须把它们排除在 deform 权重核对之外(归到"控制骨"类,见 §1)。
- 注意:`influence=0` 会让 mmd_tools 删除约束(不生成付与);付与源骨在骨数组里须排在付与子骨**之前**。
